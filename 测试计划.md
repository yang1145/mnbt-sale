## 二、部署前准备

### 2.1 环境要求

- **PHP 版本**: 8.0+ (推荐 8.1 或 8.2)
- **MySQL**: 5.7+ / MariaDB 10.3+
- **必需扩展**: `pdo_mysql`, `mysqli`, `curl`, `gd`, `openssl`, `mbstring`

### 2.2 部署步骤

1. **备份当前生产环境**
   ```bash
   # 备份代码
   tar -czf backup-$(date +%Y%m%d).tar.gz /path/to/mnbt-sale
   # 备份数据库
   mysqldump -u root -p mnbt_sale > backup-db-$(date +%Y%m%d).sql
   ```

2. **拉取最新代码**
   ```bash
   cd /path/to/mnbt-sale
   git pull origin main
   ```

3. **更新 Composer 依赖**（在服务器上执行）
   ```bash
   composer update
   # 如果没有 composer.json, 至少确保 vendor/topthink/* 包是最新的
   ```

4. **清除运行时缓存**
   ```bash
   rm -rf runtime/cache/*
   rm -rf runtime/temp/*
   rm -rf runtime/log/*
   ```

5. **检查 PHP 版本**
   ```bash
   php -v  # 确认 PHP 8.x
   ```

## 七、已知风险与回滚方案

### 7.1 已知风险

1. **vendor 包兼容性**: `vendor/topthink/*` 下的扩展包（think-captcha, think-image 等）可能需要 `composer update` 以支持 PHP 8
2. **服务器插件兼容性**: `plugins/host/` 下的服务器管理插件可能使用了 PHP 7 不兼容语法，需逐一测试
3. **cron 批量操作**: 全部删除（qbdelete）改为单条 SQL 批量删除，行为一致但失败计数逻辑略有变化
4. **cartid 显示变化**: 订单列表中 `cartid` 字段从数字 ID 变为产品名称字符串，前端模板如有依赖数字类型需检查

### 7.2 回滚方案

如测试发现严重问题，执行回滚：

```bash
# 查看提交历史
git log --oneline

# 回滚到升级前 (25b67c2 是升级前的最后一个 commit)
git revert HEAD~8..HEAD  # 撤销最近 8 个 commit (保留历史)
# 或
git reset --hard 25b67c2  # 硬回滚 (丢弃所有更改)

# 恢复数据库 (如有需要)
mysql -u root -p mnbt_sale < backup-db-YYYYMMDD.sql
```

### 7.3 渐进式部署建议

1. **先在测试环境验证**（推荐）
2. **生产环境灰度**：先部署代码，不开启 cron，手动测试核心功能
3. **开启 cron**：确认核心功能正常后，开启定时任务
4. **监控日志**：部署后 24 小时内密切监控 `runtime/log/` 下的错误日志
