# MNBT-SALE 宝塔面板安装教程

本教程基于 [MNBT-SALE 系统部署指南](../README.md)，介绍如何在宝塔面板（BT Panel）中完成本系统的安装与部署。

## 环境要求

- PHP 7.2 或更高版本
- MySQL 5.7 或更高版本
- Nginx 服务器（本教程以 Nginx 为例）
- PHP 扩展：`pdo_mysql`、`curl`、`gd`、`openssl`

> 截图中使用的环境版本为 **Nginx 1.30.3**、**PHP-7.2**、**MySQL 5.7.44**，仅供参考，实际安装时选择符合要求的版本即可。

## 一、安装宝塔面板

在服务器（CentOS/Ubuntu/Debian 等 Linux 系统）的终端中执行宝塔官方安装脚本：

```bash
if [ -f /usr/bin/curl ];then curl -sSO https://download.bt.cn/install/install_panel.sh;else wget -O install_panel.sh https://download.bt.cn/install/install_panel.sh;fi;bash install_panel.sh ed8484bec
```

安装完成后，根据终端提示的面板地址、用户名和密码登录宝塔面板。

## 二、安装运行环境

登录宝塔面板后，打开 **软件商店**，搜索并安装以下运行环境：

1. **Nginx**（推荐 1.20+）
2. **PHP-7.2** 或更高版本
3. **MySQL 5.7** 或更高版本

## 三、下载源码

从 GitHub 下载本项目的源码压缩包：

![下载源码](下载源码截图.png)

下载完成后，你将得到一个名为 `mnbt-sale.zip` 的压缩包。

## 四、添加网站

进入宝塔面板 **网站** → **PHP 项目**，点击 **添加站点**：

![添加网站 1](添加网站1.png)

在弹出的窗口中填写：

- **域名**：输入你已解析到服务器的域名（没有域名可先用 IP 地址）
- **根目录**：保持默认即可（通常为 `/www/wwwroot/你的域名`）
- **PHP 版本**：选择 **PHP-72**
- **数据库**：先选择 **不创建**，稍后我们会手动创建数据库

![添加网站 2](添加网站2.png)

点击 **提交** 完成站点创建。

## 五、上传并解压源码

在网站列表中，点击刚创建站点对应的 **根目录** 路径，进入网站目录：

![点击进入网站目录](点击进入网站目录.png)

将下载好的 `mnbt-sale.zip` 上传到该目录：

![上传源码](上传源码到网站目录.png)

上传完成后，右键点击压缩包，选择 **解压**：

![解压源码](解压源码.png)

> **注意**：如果解压后出现 `mnbt-sale` 等外层文件夹，请将其中的文件全部移动到网站根目录，确保 `app`、`public`、`runtime` 等文件夹与网站根目录同级。

## 六、设置运行目录为 public

本系统基于 ThinkPHP 框架，Web 服务器运行目录必须为 `public` 目录。

在网站列表中，点击站点右侧的 **设置**：

![设置网站相关配置](设置网站相关配置.png)

切换到 **网站目录** 选项卡，将 **运行目录** 修改为 `/public`，然后保存：

![设置网站运行目录](设置网站运行目录.png)

## 七、配置伪静态

切换到 **伪静态** 选项卡，选择规则类型或手动填入以下内容，然后保存：

```nginx
location ~* (runtime|application)/{
    return 403;
}
location / {
    if (!-e $request_filename){
       rewrite  ^(.*)$  /index.php?s=$1  last;
       break;
    }
}
```

![配置网站伪静态设置](配置网站伪静态设置.png)

## 八、创建数据库

进入宝塔面板 **数据库**，点击 **添加数据库**：

![添加数据库](添加数据库.png)

填写数据库信息：

- **数据库名**：例如 `mnbt-sale`
- **用户名**：例如 `mnbt-sale`
- **密码**：设置一个安全的密码
- **字符集**：选择 `utf8mb4`

记下数据库名、用户名和密码，稍后在安装向导中使用。

## 九、运行安装向导

在浏览器中访问你的网站域名，系统会自动跳转到安装向导。

### 1. 许可协议

阅读并勾选同意许可协议，点击 **下一步**：

![安装向导 1](安装向导1.png)

### 2. 环境检测

系统会自动检测 PHP 版本、扩展和目录写入权限。如果全部通过，点击 **下一步**：

![安装向导 2 环境监测](安装向导2环境监测.png)

### 3. 数据库配置

填写刚才创建的数据库信息：

- 数据库主机：`127.0.0.1`
- 数据库端口：`3306`
- 数据库名：`mnbt-sale`
- 数据库用户名：`mnbt-sale`
- 数据库密码：你设置的密码
- 表前缀：可保持默认（例如 `dd_`）

![安装向导 3 设置数据库](安装向导3设置数据库.png)

### 4. 管理员设置

设置网站名称和管理员账号信息：

- **网站名称**：你的站点名称
- **管理员账号**：例如 `admin`
- **管理员密码**：设置一个强密码

![安装向导 4 设置网站相关信息](安装向导4设置网站相关信息.png)

点击 **完成安装**，等待安装完成。

## 十、安装完成

安装成功后，系统会在项目根目录生成 `install.lock` 文件，防止重复安装。

访问地址：

- 前台首页：`http://你的域名/`
- 后台管理：`http://你的域名/admin`

## 安全建议

1. 将 [app/config.php](../app/config.php) 中的 `app_debug` 设置为 `false`。
2. 安装完成后建议删除 `app/install` 目录，或确保 `install.lock` 文件存在。
3. 首次登录后台后，请立即修改管理员密码。
4. 定期备份数据库和重要文件。

## 常见问题

1. **页面空白或 500 错误**
   - 检查 PHP 版本是否符合要求。
   - 确认 `runtime` 目录是否有写入权限。
   - 确认网站运行目录是否已设置为 `public`。

2. **页面样式丢失**
   - 确认运行目录为 `public`。
   - 确保 `public/static` 目录中的静态资源可以正常访问。

3. **数据库连接失败**
   - 检查数据库主机、端口、用户名、密码是否填写正确。
   - 确认 MySQL 服务是否已启动。
   - 确认数据库用户具有对应数据库的全部权限。

4. **安装向导无法访问**
   - 确认伪静态规则已正确配置。
   - 检查 `app/database.php` 和 `runtime` 目录是否有写入权限。
